Front-End used to be simple until 10-15 years ago. Now, websites are more interactive and JS is more powerful, so what it means to be a Front-End Developer has changed. May be two skillsets - (1) JS - programming and logic and making applications behave a certain way and (2) HTML & CSS how things look. Two specialties but they can interact.

DOM and JS & HTML Interaction

JS was invented after HTML and CSS, a tool to make an HTML interactive/logical.

Layer between HTML document and live memory of program - Document Object Model - a way of representing HTML inside JS code. Ex: JS Ojbect representing a tag in HTML.

Global Object that is always available in JS (e.g., console) - document is an object that lets us 'talk' to the HTML. Can use JS to manipulate your HTML document such as through "getElementById" and "innerHTML".

See Document Object on MDN. Tons of methods. Variables created by document object have a type of "document nodes."

Responsive - Mobile First Development

Easier to build something that will respond elegantly at any view. Start small.

main tool - media query - to apply conditional rules that depend on the size of the viewport.
